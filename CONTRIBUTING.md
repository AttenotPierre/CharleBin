# Guide de contribution

Merci de votre intérêt pour contribuer au projet CharleBin.

## Règles générales
- Toute modification doit être réalisée sur une branche dédiée.
- Le code doit être clair, lisible et cohérent avec l’existant.
- Les commits doivent être explicites et porter sur un seul sujet.

## Processus de contribution
1. Vérifier que vous êtes sur la branche `main`.
2. Créer une nouvelle branche à partir de `main`.
3. Effectuer les modifications nécessaires.
4. Créer un ou plusieurs commits clairs et cohérents.
5. Pousser la branche sur le dépôt distant.
6. Ouvrir une Pull Request sur GitHub.
7. Attendre la review avant toute fusion.

## Pull Requests
- Une Pull Request doit adresser un seul problème.
- Elle doit être testable en local.
- Elle ne doit pas contenir de code mort ou de commentaires inutiles.
- La Pull Request ne doit pas être mergée sans validation.

## Tests
- Les tests doivent être lancés avant l’ouverture d’une Pull Request.
- Toute Pull Request cassant les tests sera refusée.
